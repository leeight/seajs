为什么使用 SeaJS ？

SeaJS 追求简单、自然的代码书写和组织方式，具有以下核心特性：

  • 简单友好的模块定义规范：SeaJS 遵循 CMD 规范，比 AMD 更简单友好，可以像 Node.js 一般书写模块代码。
  • 自然直观的代码组织方式：模块依赖的自动加载、配置信息的简洁清晰，可以让我们更多地去享受编码的乐趣。

SeaJS 还提供常用插件，非常有助于开发调试和性能优化，并具有丰富的可扩展接口。

10 分钟上手 SeaJS

这是个小游戏，调皮的字母来自神秘的大海深处。当鼠标轻轻滑过时，字母会旋转到正确位置。
注意：该例子仅在高级浏览器下有效，推荐用 Chrome 浏览。
来试试吧，看你能否让所有字母都听话……

下面花 10 分钟，来看看如何来实现这个小游戏。

目录结构

先下载源码包：hello-seajs.zip

解压后可看到目录结构：

hello-seajs/
  |-- assets
  |     |-- main
  |     `-- sea-modules
  `-- index.html

assets 目录存放 css、js 等静态资源文件，其中 sea-modules 下存放的是我们需要用到的第三方模块，可通过 spm install 来获取：

$ cd hello-seajs/assets
$ spm install seajs
$ spm install gallery.jquery

注意：

  1. 安装的 jquery.js 是经过封装的模块文件，不能用 jQuery 官网提供的原始文件。
  2. spm 是包管理工具，我们先了解有这么一个工具就好，不用着急。

最重要的一个目录是 main，用来存放模块源码：

main/
  |-- docs
  |-- examples
  |-- src
  |    |-- main.css
  |    |-- main.js
  |    `-- spinning.js
  |-- tests
  |-- package.json
  `-- README.md

这是一个模块的标准目录结构，docs 目录用来存放文档，examples 存放示例，src 存放源码，tests 存放测试用例。

package.json 是模块信息文件，指明模块名、版本等基本信息，还包括 spm 的打包配置等信息。

在页面中加载模块

在 index.html 里，首先需要引入 sea.js，然后通过 use 方法来加载模块： 

<script src="assets/sea-modules/seajs/1.3.0/sea.js"></script>
<script>
  seajs.config({
    alias: {
      'jquery': 'gallery/jquery/1.8.2/jquery' // 指定使用的 jQuery 版本
    }
  })

  // 加载入口模块
  seajs.use('./assets/main/src/main')
</script>

上面是最简单的一种写法，更多加载方式请参考：模块加载器

书写模块代码

这个小游戏的核心文件是 spinning.js： 

define(function(require, exports, module) {

  // 获取依赖，spinning 只依赖 jquery
  var $ = require('jquery')

  // 下面是这个小游戏的核心代码，有兴趣的可以等看完这篇文档后再去阅读
  function Spinning() { ... }

  Spinning.prototype.render = function() { ... }

  ...

  // 对外提供接口
  module.exports = Spinning
})

main.js 很简单： 

define(function(require) {

  // 得到 Spinning 函数类
  var Spinning = require('./spinning')

  // 初始化
  var s = new Spinning('#container')
  s.render()

})

通过 require 引入依赖，通过 exports 提供接口，一切就这么简单。更多用法请参考：CMD 模块定义规范

打包部署

对于正式项目，在发布上线前，还需要对源码进行压缩、合并等操作。在 SeaJS 里，通过包管理工具 spm 来实现。

安装好 spm 后，执行以下命名： 

$ cd hello-seajs/assets/main
$ spm build 
...
BUILD SUCCESS!
$

这样就打包好了，生成的文件在 sea-modules 目录下： 

sea-modules/
  |-- hello-seajs
        `-- main
              `-- 1.0.0
                    |-- main-debug.css
                    |-- main-debug.js
                    |-- main.css
                    `-- main.js

其中 main.js 中已经合并了 spinning.js 文件。详尽用法请参考：spm Wiki

打包好后，将需要的文件，比如 sea-modules 同步到线上，然后修改 index.html 中的引用路径： 

<link rel="stylesheet" href="http://yoursite.com/assets/sea-modules/hello-seajs/main/1.0.0/main.css">
...
<script src="http://yoursite.com/assets/sea-modules/seajs/1.3.0/sea.js"></script>
<script>
  seajs.config({
    alias: {
      'jquery': 'gallery/jquery/1.8.2/jquery' // 指定使用的 jQuery 版本
    }
  })

  // 加载入口模块
  seajs.use('hello-seajs/main/1.0.0/main')
</script>

修改好后，发布上线，就可以回家睡觉或出国旅游了……

注意：在实际项目中，不需要修改 index.html 中的引用。可以通过配置，自动适配环境。比如可以： 

  // 加载入口模块
  seajs.use(location.host === 'localhost' ? './assets/main/src/main' : 'hello-seajs/main/1.0.0/main')

非静态页面时，一般通过变量来配置，比如 PHP 可以：

  // 加载入口模块
  seajs.use('<?=$MAIN_SRC?>')
